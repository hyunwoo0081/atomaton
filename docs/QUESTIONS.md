# 프로젝트 구현을 위한 질문 목록

안녕하세요! `PROJECT_CHECKLIST.md`와 기획서를 다시 한번 검토했습니다. 개발을 명확하게 진행하기 위해 몇 가지 질문을 드립니다.

### **1. 일반**

- **프로젝트 이름:** 현재 폴더명인 `make-clone`을 공식 프로젝트 이름으로 사용해도 될까요?
프로젝트 이름을 Atomaton 이라고 해주면 돼. 폴더 이름은 내가 추후에 변경할게

### **2. 사용자 및 권한**

- **개발자 계정:** 사용자를 '개발자 계정'으로 지정하는 방법은 무엇인가요? 데이터베이스에 특정 필드(`is_developer=true` 등)를 추가하는 방식일까요, 아니면 별도의 등록 절차가 있나요?
데이터 베이스에 column 추가해서 개발자 계정을 표현해주면 돼. 개발자가 적을 예정이고 유저도 그리 많진 않을 예정이라 이렇게 설계해줘도 괜찮을 것 같아.


### **3. 트리거 (Triggers)**

- **IMAP Polling 주기:** 사용자가 설정할 수 있는 최소/최대 Polling 주기는 몇 분인가요? 그리고 사용자가 설정하지 않았을 경우의 기본값은 몇 분으로 할까요?
  - 기본 값은 30분에 1번이고, 최소 값은 1분에 한 번이 최소가 되도록 해주면 좋을 것 같아. 
- **Webhook API 키:** Webhook 인증에 사용되는 API 키는 사용자(계정) 단위로 발급되나요, 아니면 각 Webhook 트리거마다 개별적으로 발급되나요?
- API 키는 웹 훅마다 발행되도록 해주면 돼. webhook 을 발행할 때, api 주소는 프로젝트 id 가 해쉬형태로 표현돼서 url 에 포함되도록 만들어주면 돼.

### **4. 액션 (Actions)**

- **Discord 메시지 형식:** Discord로 전송되는 알림 메시지의 내용은 사용자가 직접 설정할 수 있나요? 아니면 "새 이메일: [제목] - [보낸사람]" 과 같이 정해진 형식이 있나요? 만약 형식이 정해져 있다면, 어떤 정보(예: 제목, 보낸사람, 내용 일부)를 포함해야 할까요?
- 너가 말해준 정보들을 자유로운 형태로 넣을 수 있도록 만들어주면 돼. 해당 Action까지 오기 위해서 지나쳤던 Trigger 와 Action 에 나오는 모든 정보를 해당 Action 에서 사용할 수 있도록 만들어주면 될 것 같아. 
- **Notion 액션 상세 내용 (가장 중요):** Notion 액션의 구체적인 동작을 정의해야 합니다.
    - **어떤 작업**을 원하시나요? (예: 데이터베이스에 페이지 생성, 기존 페이지에 콘텐츠 추가, 데이터베이스 아이템 속성 변경)
    - 사용자는 이 작업을 어떻게 설정하나요? 예를 들어, "데이터베이스에 페이지 생성"이라면, 사용자가 어떤 데이터베이스 ID에 생성할지, 그리고 트리거 데이터(예: 이메일 제목)를 페이지의 어떤 속성(예: '이름' 속성)에 매핑할지 설정하는 UI가 필요한가요?
    - 일단 가장 급하게 필요한건, database 에 row를 추가하고 데이터베이스를 변경할 수 있도록 하는게 가장 중요해. 그리고, 작업을 설정하려면 UI를 통해서 하도록 설계하고 싶어. 이거 매핑하는 방식은 UI로도 설정이 가능하도록 해주면 좋을 것 같고, 간단한 스크립트 코드도 가능하도록 만들어주면 좋을 것 같아. 너무 어려우면, 추후 작업에 포함시켜주면 좋겠어.
### **5. 로그 및 데이터**

- **로그 보존 정책:** "최근 1,000건, 3일 보존" 정책은 전체 시스템에 대한 글로벌 기준인가요, 아니면 사용자 계정별 기준인가요?
- 로그 보존 정책은 사용자 계정별 기준이야. 추후 최대 보존기간을 조절할 수 있도록 할 예정이니 (지금은 3일이고) 이 부분에서 확장성 있도록 설계해줘.

### **6. 프론트엔드 (Frontend)**

- **워크플로우 UI:** 'Make 스타일'의 UI 구현을 위해, `React Flow` 라이브러리를 사용하는 것에 동의하시나요? 초기 버전에서는 더 간단한 목록 형태로 구현한 후 점진적으로 시각화하는 방식도 괜찮을까요?
- 네, 보안상 문제가 있는 버전이 아니라면 라이브러리 사용해도 됩니다.
- **개발자 대시보드:** 개발자용 '전체 Trigger 실행 현황' 화면에 어떤 정보들이 표시되어야 할까요? (예: 트리거 ID, 계정 ID, 마지막 실행 시각, 상태, 에러 메시지 등)
- 어떤게 실행되었는지, Action 이 어느정도 있는지, 실제 Action은 얼마나 실행이 되었는지, Trigger 는 어떻게 설정이 되어있고, 얼마나 성능을 차지하고 있고 실행시간은 얼마이고 이런 부분을 추후에 개선할 수 있도록 모니터링을 하는게 목적이라, 개발자의 입장에서 matric 을 잘 보여줄 수 있도록 만들어주면 좋을 것 같아. 특히나 너무 오래 걸리는데, trigger 가 짧게 설정되어 있다던지, 서버 부하를 너무 많이 먹는다던지 이런 부분은 나중에 봐서 내가 개선을 해야할 사항이니까 이런 정보에 대해서 쉽게 모니터링 할 수 있도록 만들어주면 좋을 것 같아.


위에 대한 답변을 주시면, `PROJECT_CHECKLIST.md`를 최종 업데이트하고 본격적인 개발을 시작하겠습니다.
