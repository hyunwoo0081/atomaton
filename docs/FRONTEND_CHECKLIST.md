# 자동화 에디터 UI 기능 개선 체크리스트

현재 UI는 기본적인 `reactflow` 캔버스와 노드 추가 기능만 제공하고 있어 제어할 수 있는 부분이 부족합니다. 아래 체크리스트를 통해 프론트엔드 UI의 제어 기능을 강화할 수 있습니다.

## 1. 기본 노드/엣지 상호작용 강화

- [x] **노드 삭제 기능 추가**
    - [x] 선택된 노드를 삭제할 수 있는 버튼 (예: 속성 패널 내) 또는 키보드 `Delete` 키 이벤트 처리
    - [ ] 노드 우클릭 시 컨텍스트 메뉴에 '삭제' 옵션 추가
- [x] **엣지 삭제 기능 추가**
    - [x] 선택된 엣지를 삭제할 수 있는 키보드 `Delete` 키 이벤트 처리
    - [ ] 엣지 우클릭 시 컨텍스트 메뉴에 '삭제' 옵션 추가
- [ ] **노드 복사/붙여넣기 기능**
    - [ ] 선택된 노드를 복사하고 붙여넣을 수 있는 기능 (키보드 단축키 또는 컨텍스트 메뉴)

## 2. 속성 패널 기능 및 데이터 바인딩 개선

- [ ] **모든 노드 속성 데이터 바인딩 완료**
    - [ ] **Trigger (IMAP):** 호스트, 사용자, 비밀번호 입력 필드의 값을 `selectedNode.data`에 저장하고 불러올 수 있도록 연결
    - [ ] **Trigger (Webhook):** Webhook URL 생성 및 표시, 복사 버튼 기능 구현
    - [ ] **Condition:** 필드, 연산자, 값 입력 필드의 값을 `selectedNode.data`에 저장하고 불러올 수 있도록 연결
    - [ ] **Action (Discord):** Webhook URL, 메시지 내용 입력 필드의 값을 `selectedNode.data`에 저장하고 불러올 수 있도록 연결
    - [ ] **Action (Notion):** API Key, Database ID, 속성 매핑 필드의 값을 `selectedNode.data`에 저장하고 불러올 수 있도록 연결
- [ ] **데이터 유효성 검사 및 피드백**
    - [ ] 각 입력 필드에 대한 기본적인 유효성 검사 (예: 필수 값, URL 형식 등) 및 사용자에게 오류 피드백 제공
- [x] **속성 패널 닫기/열기 기능**
    - [x] 속성 패널을 닫거나 열 수 있는 UI (예: X 버튼)

## 3. 캔버스 기능 확장 및 사용자 경험 개선

- [x] **드래그 앤 드롭으로 노드 추가**
    - [x] 좌측 사이드바의 버튼을 캔버스 위로 드래그하여 노드를 추가할 수 있도록 구현 (`reactflow`의 `onDragOver`, `onDrop` 이벤트 활용)
- [ ] **캔버스 컨텍스트 메뉴**
    - [ ] 캔버스 빈 공간을 우클릭했을 때, '새 노드 추가' 등의 옵션을 제공
- [ ] **자동 레이아웃 기능 (선택 사항)**
    - [ ] 노드들이 복잡해질 경우, 자동으로 노드들을 정렬해주는 기능 (예: `dagre` 라이브러리 활용)
- [x] **실행 플로우 시각화 개선**
    - [x] 실행 결과에 따라 노드나 엣지의 색상을 변경하여 성공/실패를 시각적으로 표시 (커스텀 노드 도입으로 기반 마련)
    - [ ] 노드 위에 작은 아이콘이나 뱃지로 상태 (실행 중, 성공, 실패) 표시

## 4. 상위 레벨 기능 구현

- [ ] **자동화 저장 기능 구현**
    - [ ] "Save" 버튼 클릭 시 현재 `nodes`와 `edges` 상태를 JSON 형태로 직렬화하여 백엔드 API로 전송
    - [ ] 저장 성공/실패 피드백 제공
- [ ] **자동화 불러오기 기능 구현**
    - [ ] 기존 자동화를 편집할 때, 백엔드에서 `nodes`와 `edges` 데이터를 불러와 캔버스에 렌더링
- [ ] **자동화 활성화/비활성화 (ON/OFF) 토글**
    - [ ] 헤더 또는 별도 UI에 자동화의 활성화 상태를 변경할 수 있는 토글 스위치 구현
- [ ] **테스트 실행 기능 구현**
    - [ ] "Deploy" 버튼을 "Test Run" 등으로 변경하고, 현재 구성된 플로우를 백엔드에서 테스트 실행할 수 있는 기능 연동
    - [ ] 테스트 실행 결과(로그)를 UI에 표시

## 5. UI/UX 개선

- [ ] **로딩 상태 표시**
    - [ ] 데이터 저장/불러오기 등 비동기 작업 시 로딩 스피너 또는 메시지 표시
- [ ] **빈 캔버스 가이드**
    - [ ] 새 자동화 생성 시 캔버스가 비어있을 때, "좌측에서 노드를 드래그하여 시작하세요"와 같은 안내 메시지 표시
- [ ] **키보드 접근성**
    - [ ] 키보드만으로도 노드 선택, 이동, 삭제 등 주요 기능을 사용할 수 있도록 개선
